# Story 021: Todo And Memory Model

## User Story
As the user, I want Donna to remember todos, facts, meetings, and preferences, so that it can organize my work-life over time.

## Acceptance Criteria
- Donna stores todos with status, source, timestamps, optional due date, related people, and related topic.
- Donna stores memories with type, content, source, confidence, importance, and timestamps.
- Donna stores meeting memories extracted from chat or calendar context.
- Donna can mark todos done, dismissed, snoozed, or stale.
- Donna can forget memories on request.
- Memories can expire or decay over time where configured.

## Notes
- Raw Donna chat remains ephemeral even when structured memories are saved.
