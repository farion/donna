# Story 036: User Documentation

## User Story
As the user, I want Markdown documentation for Donna, so that I can install, configure, authenticate, and use the assistant.

## Acceptance Criteria
- Documentation lives in `docs/`.
- Main usage entry point is `docs/usage.md`.
- Documentation explains starting Donna with `donna`.
- Documentation explains auth mode with `donna --auth`.
- Documentation explains AI provider setup.
- Documentation explains Microsoft Graph authentication.
- Documentation explains config path `~/.config/donna/donna.toml`.
- Documentation explains task definitions in `~/.config/donna/tasks/`.
- Documentation explains Markdown task prompts.
- Documentation explains `/exit`, `/hide`, and `/changechar [name]`.
- Documentation explains Tab model switching.
- Documentation explains avatar states.
- Documentation explains that Donna chat is ephemeral.
- Documentation explains stored data, attention behavior, approvals, Obsidian integration, and privacy boundaries.
- Documentation includes example config and task files.

## Notes
- Documentation should be clear for a technically comfortable single user.
